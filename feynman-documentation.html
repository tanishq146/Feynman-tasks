Section 01

Platform Overview & Architecture

Feynman is a cognitive platform that treats your knowledge as a living entity. It visualizes your mind as a 3D neural brain, applies real-time memory decay, and provides AI-driven tools to ensure true conceptual ownership rather than passive recall.

Core Philosophy

Named after Richard Feynman's principle — "If you can't explain it simply, you don't understand it well enough" — the platform enforces deep understanding through its five-stage knowledge lifecycle: Ingestion → Analysis → Decay → Reinforcement → Expansion.

Section 02

The 3D Neural Brain

The central visualization of the Feynman platform. Every piece of knowledge you ingest becomes a luminous node orbiting within a three-dimensional brain model, organized by functional brain region.

Knowledge Nodes
Each node is a 3D sphere rendered with MeshPhysicalMaterial featuring iridescence, clearcoat, and transmission properties. Node size reflects importance. Node color and glow intensity reflect current memory strength — healthy nodes glow brightly, fading nodes dim, and critical nodes pulse red.
Neural Threads
Connections between related knowledge nodes are rendered as volumetric 3D tubes (not flat lines) with pulsing animations. Each thread has an invisible wide hit-mesh (radius 0.08) for reliable click detection. Clicking a thread opens the Connection Panel showing the AI-generated relationship rationale.
Ingestion Particles
When new knowledge is added, animated particles stream from the input bar along a trajectory toward the target brain region, providing visceral visual feedback that knowledge is being "absorbed" into the brain.
Environment & Lighting
The 3D scene uses a "night" Environment preset for consistent, renderer-safe visuals across browsers. Ambient and directional lighting ensure node materials render with premium reflections and depth.

Knowledge Ingestion & Lobe Mapping

Raw text enters the system through the InputBar. AI analyzes the concept, classifies the broad topic, and maps it to a specific functional brain lobe — determining where the knowledge lives spatially in your 3D brain and how quickly it will naturally decay.

The InputBar

The primary ingestion interface. Users type or paste any concept, fact, or insight. The AI processes it in real-time through:

  • Topic classification (Science, History, Philosophy, etc.)

  • Automatic connection detection to existing knowledge nodes

Section 04

The Ebbinghaus Decay System

Feynman simulates memory decay in real-time. Once created, every knowledge node begins to "forget" — its strength is recalculated every 10 seconds using an exponential forgetting curve, making the decay visible before your eyes.

Ebbinghaus Forgetting Formula

strength = 100 × e−(decayRate × daysSinceReview)

Where decayRate is determined by the node's brain lobe assignment, and daysSinceReview is the time elapsed since the user last reviewed or reinforced the memory.

Memory Strength States

StateStrength RangeVisual IndicatorSystem Behavior
Healthy70–100%Bright glow, full luminosityNo intervention needed
Fading30–69%Dimmed glow, amber tintFadingWarning appears for reinforcement
Critical10–29%Pulsing red, flickeringUrgent study prompt activated
Forgotten0–9%Nearly invisibleNode at risk of being lost

The Decay Ticker

The useDecayTicker hook runs a client-side interval every 10 seconds, recalculating the strength of every node in the user's brain. This ensures the 3D visualization reflects real-time memory decay — nodes literally dim and change color as time passes. The hook also provides utility functions:

  • getForgettingCurveData() — Projects decay trajectory for visualization charts
  • timeUntilThreshold() — Human-readable countdown to when a node will enter "fading" or "critical" state

Fading Warning System

When nodes enter fading or critical state, the FadingWarning component appears as an ambient notification. It counts the number of at-risk nodes and provides a direct one-click path into Study Mode, pre-filtered to show only the fading nodes that need attention.

Section 05

The Feynman Panel — Reinforcement Suite

The primary interaction hub for deepening understanding. Triggered when clicking a knowledge node in the 3D brain, this panel provides AI-driven tools that test for genuine comprehension — not simple recall.

Feynman Analysis
Before any reinforcement begins, the system generates three foundational layers for every node:
  • Simplified Explanation — How would a child understand this concept?
  • Key Insight — The most critical logic needed to grasp the concept
  • Connection Logic — How does this relate to neighboring knowledge nodes?
The Challenge
A synthesized logic probe that prevents simple "I remember the definition" responses. The AI (Groq Llama 3) grades answers on a 0–100 scale with five verdict levels:
  • Master (90–100) — Complete conceptual ownership
  • Understands (70–89) — Solid grasp with minor gaps
  • Partial (50–69) — Key pieces missing
  • Memorized (30–49) — Recall without true understanding
  • Missing It (0–29) — Fundamental misunderstanding

Success resets the node's current_strength back to 100, combating natural decay.

Teach-It Simulation
A Socratic teaching mode where the user explains the concept in simple terms to a virtual AI child. The AI evaluates for:
  • Use of jargon (penalized)
  • Over-complication (penalized)
  • Technical accuracy (required)
  • Clarity of analogies (rewarded)

Earning "Feynman Certification" permanently reduces the node's decay rate by 10%, making the memory structurally stickier over time.

Knowledge Gap Discovery
During analysis, the AI identifies related concepts that are "logically missing" from the user's current knowledge graph. These gaps are presented as interactive teasers. Clicking "Fill Gap" automatically ingests a new node and builds a neural connection to the source node.
Real-Life Moment
A hyper-contextualized "mission" generated by the AI for the user to apply their knowledge in their own life within the next week. This bridges the gap between theoretical understanding and practical application, ensuring learning transcends the screen.
Section 06

Node Dive — Deep Knowledge Exploration

Triggered by double-clicking a knowledge node. A full-screen immersive experience for cognitive deep-dives, offering two distinct interaction modes.

Strict Mode
The AI acts as an expert examiner. It challenges every claim the user makes, demands evidence for assertions, and identifies logical inconsistencies. This mode is designed to stress-test understanding and reveal hidden knowledge gaps. The AI will not accept vague or surface-level explanations.
Non-Strict Mode
A conversational exploration mode. The AI acts as a collaborative partner, helping the user explore the concept from multiple angles, draw connections to other knowledge, and discover new perspectives. Ideal for creative exploration and building mental models.

Interaction Mechanics: A 300ms click timer resolves the conflict between single-click (opens Feynman Panel) and double-click (triggers Node Dive). This ensures precise, frustration-free navigation between the two features.

Section 07

Lobe View — Regional Brain Exploration

A dedicated 3D interface for exploring knowledge within a specific brain lobe. Activated by clicking a lobe indicator in the brain visualization, it zooms into that region and displays only the nodes assigned to that functional area.

Lobe View provides a focused, region-specific environment where users can:

  • See all knowledge nodes within a specific brain region (Hippocampus, Prefrontal Cortex, etc.)
  • Visualize the connections between nodes in that region
  • Identify which knowledge is healthy, fading, or critical within each domain
  • Interact with individual nodes for reinforcement
  • Understand the topological density of their knowledge in specific fields

The transition into Lobe View is handled by the LobeDiveTransition component, which provides a smooth animated zoom effect from the full brain to the target lobe.

Section 08

Neural Threads — Knowledge Connections

The connective tissue of the knowledge brain. Neural threads represent AI-detected relationships between knowledge nodes, visualized as volumetric 3D tubes with pulsing energy animations.

When a user clicks an existing neural thread, the Connection Panel opens, providing:

  • Relationship Rationale — AI-generated explanation of why these two concepts are connected
  • Connection Strength — Visual indicator of how strong the conceptual link is
  • Source & Target Details — Quick access to both connected nodes

Technical Detail: Threads are rendered as THREE.TubeGeometry with THREE.MeshPhysicalMaterial. An invisible wide hit-mesh (radius 0.08) wraps each thread to ensure reliable click detection even on thin connections. This prevents frustrating miss-clicks on mobile and desktop.

Section 09

Feynman AI Chat — Your Second Brain

A conversational AI interface that knows everything in your brain. Feynman AI references your stored knowledge nodes when answering questions, creating a personalized AI assistant grounded in your own learning.

Knowledge-Grounded Responses
Every response explicitly references which nodes from your brain were used to construct the answer. These appear as clickable "FROM YOUR BRAIN" badges at the bottom of each message, creating full transparency about the AI's reasoning chain.
Why Chain Mode
An advanced introspection tool. When enabled, after the AI answers any question, it enters a 5-level "Why Chain" — progressively deeper questions that drill down to first principles:
  • Each level asks "why?" about the previous answer
  • Progress visualized with a depth indicator (1/5 → 5/5)
  • After 5 levels, generates a comprehensive conceptual analysis
  • Can be exited at any point by pressing "End"
Rich Markdown Rendering
All AI responses render with full markdown support: headers, bold/italic, inline code, ordered and unordered lists, horizontal rules, and links — all styled with the Feynman neural aesthetic.
Compact & Expanded Views
The chat panel supports two layouts: a compact sidebar mode (360px) overlaying the brain, and a full-screen expanded mode (800px centered). Toggle between them with the expand/collapse button. On mobile, it defaults to full-screen.
Section 10

Study Mode — Adaptive Flashcards

An AI-driven adaptive flashcard system that prioritizes fading and critical knowledge nodes. Each session dynamically generates questions based on the user's current memory strengths, ensuring study time is spent where it matters most.

Study Mode features:

  • Priority Ordering — Nodes closest to critical state appear first
  • Pre-filtered Sessions — Can be launched from FadingWarning with only at-risk nodes
  • AI-Generated Questions — Each flashcard question is uniquely synthesized per session, preventing memorization of fixed question sets
  • Session Tracking — Records completion rates, scores, and time spent across sessions
  • Strength Reset — Successfully answering a flashcard resets the node's strength, pushing back decay
Section 11

Notes Workspace — Rich Knowledge Documentation

A full-screen note-taking workspace inspired by Obsidian, built directly into Feynman. Write freely, then let AI extract knowledge nodes from your notes — bridging unstructured thinking and structured knowledge.

Rich Text Editing
Unlimited notes with sidebar navigation, markdown-style formatting, auto-save with debounced persistence (1-second delay), and pin/unpin capability to keep important notes at the top.
Image Upload
Supports both drag-and-drop and button-based image uploads. Inline images are resizable (drag handles), zoomable (click to open full), and removable. Images flow naturally in the document like Obsidian.
Voice Recording
Full voice note recording using the Web Audio API with professional audio settings:
  • Echo cancellation and noise suppression enabled
  • Auto gain control for consistent volume
  • 48kHz sample rate for voice clarity
  • 128kbps bitrate with Opus codec support
  • Custom audio player with seek bar, volume control (up to 200%), and renameable voice notes
AI Knowledge Extraction
The "Analyze with AI" button sends note content to Gemini, which suggests new knowledge nodes based on the user's writing. Each suggestion includes a title, content summary, and category. Users approve or dismiss each suggestion — approved suggestions are ingested directly into the 3D brain as new nodes.
Markdown Export
Notes can be downloaded individually as markdown files, preserving formatting and structure for use in external tools like Obsidian, Notion, or any markdown editor.

Mind Mirror — Journaling & Introspection

A dedicated journaling interface for emotional and psychological introspection. Unlike the Notes Workspace (which is for knowledge documentation), Mind Mirror is designed for raw, unfiltered self-reflection that feeds into the Mindscape emotional analysis system.

Mind Mirror provides multiple introspection modes that help users explore their inner landscape:

  • Free Writing — Stream-of-consciousness journaling with no constraints
  • Guided Prompts — AI-generated reflection prompts tailored to the user's emotional state
  • Emotional Tagging — Journal entries feed into the Mindscape's emotional agent system
  • Historical Review — Browse past entries to track emotional patterns over time

Journal entries are processed by the Mindscape backend to update emotional agent states, contributing to the Mirror Report and the living mind graph.

Section 14

Mindscape — The Living Mind Graph

A full-viewport 2D mind graph paired with a 26-agent emotional simulation engine. Mindscape maps your psychological landscape — not your knowledge, but your emotional architecture. It is the emotional counterpart to the 3D knowledge brain.

Mind Graph
A force-directed graph that maps psychological constructs — thoughts, beliefs, emotional patterns — as interconnected nodes. Supports filtering by type, timeline cutoff for temporal analysis, and history mode for reviewing past snapshots of your psychological state.
26 Emotional Agents
The Agent Simulation panel houses 26 distinct emotional agents, each representing a specific psychological force. These agents process your journal entries and knowledge to calculate dominance scores, creating a real-time emotional profile.
  • Core 8 (Plutchik Primary) — Sentinel (Fear), Fury (Anger), Euphoric (Joy), Mourner (Sadness), Believer (Trust), Purist (Disgust), Oracle (Anticipation), Witness (Surprise)
  • Self-Conscious — Phantom (Guilt), Exile (Shame), Crown (Pride)
  • Social — Mirror (Envy), Hollow (Loneliness), Bridge (Empathy), Garden (Gratitude)
  • Anticipatory — Void (Anxiety), Torch (Hope), Ghost (Regret)
  • Complex Dyads — Judge (Contempt), Hearth (Love), Sublime (Awe), Abyss (Despair)
  • Behavioral — Wanderer (Curiosity), Anchor (Nostalgia), Spark (Frustration), Drift (Boredom)
Dominance Visualization
Two visual representations of emotional agent dominance:
  • Arc Chart — A floating SVG ring in the bottom-left of the graph showing proportional dominance of each active agent
  • Mini Bar — A compact horizontal bar in the top navigation showing top 8 agents at a glance
Agent Memory Panel
Each emotional agent maintains its own memory — a history of the journal entries, thoughts, and knowledge that influenced its dominance score. This panel provides transparency into why specific agents are active or dormant.
Mirror Report
A periodic psychological synthesis generated by the AI. The Mirror Report analyzes your emotional agent state trajectories and mind graph topology to produce a narrative insight report. New reports within 24 hours trigger a notification dot on the Mirror tab. Full-screen reading mode available.
Agent Dialogue Overlay
During active simulations, emotional agents visually debate on the graph canvas itself. Each agent appears as a node in a neural constellation pattern, with speech bubbles showing their contributions. The currently speaking agent is highlighted.
God View Controls
A toolbar for graph manipulation: filtering nodes by type, adjusting timeline cutoff, triggering agent simulation, toggling history mode, and browsing historical graph snapshots to see how your mind has changed over time.
Node Sidebar
Clicking any node in the mind graph opens a detailed sidebar showing the node's content, its connections to other nodes, and a resolve/unresolve toggle for tracking the status of psychological conflicts or questions.
Section 15

Delphi — Social Simulation Oracle

Simulate your people before you face them. Delphi lets you build AI-driven personality models of real people in your life, then simulate how they would react to any scenario — before you have the actual conversation.

Council Management
Build a "council" of AI personality agents representing people in your life. Two creation methods:
  • Quick Add — Name, relationship, and description in one form
  • Build via Chat — Conversational agent builder where you describe the person through dialogue with Delphi, who asks probing questions to build a richer personality model

Each agent is assigned a unique color, initials, and relationship tag. Agents can be deleted from the council at any time.

Multi-Agent Scenario Simulation
Type any scenario (e.g., "I want to drop out of college and pursue my startup") and Delphi simulates how your council members would react. The simulation runs in multiple conversation rounds:
  • Round 1 — Initial Reactions: Each agent responds independently to the scenario
  • Round 2+ — Cross-Discussion: Agents respond to each other, creating realistic group dynamics
  • Messages stream at a readable pace with typewriter animation (38ms per character)
  • Round dividers mark the transition between initial reactions and cross-discussion
Post-Simulation Intelligence
After the simulation completes, three analysis layers appear:
  • Insights — Key takeaways with color-coded severity (red, green, blue, amber)
  • Outcome Probability — Calibrated percentage predictions for possible outcomes with sentiment indicators (positive, negative, neutral)
  • Trigger Map — Per-person language intelligence showing what phrases to avoid and what to say instead
Rehearsal Mode
After observing the simulation, the user can enter the conversation themselves. Type what you would actually say, and each council member responds in real time based on their personality model and the established conversation context. This creates a safe practice ground for difficult conversations.
Post-Conversation Debrief
After having the real conversation, return to Delphi and describe what actually happened. The system compares its predictions with reality, generating:
  • Accuracy Score — 0–100% rating with color coding (green ≥70%, amber ≥40%, red <40%)
  • Aspect Comparison — Per-aspect breakdown of what was predicted versus what actually happened
  • Agent Model Updates — Learnings that sharpen the personality models for future simulations
Simulation History
All past simulations are stored and replayable. The History tab shows a chronological list with scenario previews, insight headlines, debrief status indicators, and one-click replay functionality.

Alexander — aMCC Willpower Forge

Not a tracker. A forge. Alexander is built on the neuroscience of the anterior midcingulate cortex (aMCC) — the brain region that grows only through resistance. This system makes the process of doing what you genuinely don't want to do visceral and rewarding.

Neural Canvas Visualization
A living, breathing canvas animation representing your aMCC growth. Features:
  • Performance-optimized particle system (70 particles desktop, 35 mobile)
  • Batched draw calls with three alpha-bucket connection rendering
  • Core glow that expands as total points increase
  • Pulse effect triggered on every task completion
  • Growth-scaled particle size and connection density
Task System
Create "resistance tasks" — things you're actively avoiding. Each task has:
  • Resistance Rating (1–5) — From "Mild" to "Pure Hell," rated before starting — not after
  • Domain Category — Body, Mind, Social, Career, or Fear
  • Lock-In Mechanic — Before completing, you must first "Lock" the task (800ms sweep animation), sealing the resistance level
  • Avoidance Multiplier — Tasks avoided for 3+ days earn bonus points (1.5×, 2×, etc.)
  • Streak Tracking — Consecutive completions tracked with a fire emoji counter
  • Point Calculation — Base points = Resistance × 25, multiplied by avoidance bonus

Growth Stages

Your aMCC evolves through five neuroplasticity stages based on total accumulated resistance points:

Additional Features

Resistance Ring
A half-circle SVG arc showing progress within the current stage, with stage markers and a pulsing glow at the current position.
Activity Log
A chronological ledger of every completed resistance task — "proof of who you're becoming." Shows points earned, task title, resistance level, and relative time.
Journey Panel
A milestone progression view showing all five stages with progress bars, completion status, and descriptive text. Active stage shows a live progress bar with current score.
Completion Burst
Visual celebration animation when a task is completed — large point value appears center-screen with a "aMCC Growth" label, accompanied by a green glow and scale animation.
Animated Counter
Total score counter that animates between values using requestAnimationFrame and direct DOM manipulation (no React re-renders), maintaining a smooth, physically-correct count-up/down animation.

Persistence: Alexander data is stored per-user in localStorage, scoped by Firebase UID. This includes all tasks, the activity log, and total points. Data persists across sessions and migrates cleanly when users switch accounts.

Engram — The Living Thinking Graph

A tool for externalizing and mapping your thinking process. Unlike the main brain (which stores factual knowledge) and Mindscape (which maps emotions), Engram captures the evolution of your thoughts across conversations — extracting what matters from your AI interactions and building a connected thinking graph.

Thought Graph (Canvas 2D Engine)
A custom-built force-directed graph renderer using native HTML5 Canvas 2D API. This dependency-free implementation was specifically chosen to avoid WebGL context conflicts with the 3D brain visualization. Features:
  • Physics-based force simulation with attraction, repulsion, and centering forces
  • Interactive pan and zoom with smooth animation
  • Maturity-colored nodes (seed → sprouting → growing → mature → evolved)
  • Click-to-select nodes with detailed sidebar view
Conversation Ingestion
Paste any AI conversation into the IngestPanel. The backend (Groq Llama 3.3 70B) extracts discrete thoughts mentioned in the conversation, identifies relationships between them, and builds a connected graph of your thinking. The AI distinguishes between:
  • Core insights vs. supporting details
  • Questions vs. conclusions
  • Contradictions vs. refinements
AI Chat Import Pipeline
A universal importer supporting direct file upload from major AI platforms:
  • Claude — JSON conversation exports
  • ChatGPT — JSON conversation exports
  • Gemini — JSON conversation exports

The 3-step import flow: Upload file → Select conversations from the parsed list → Process selected conversations into the Engram graph. Each platform's export format is normalized into a unified internal structure by the importer.js service.

Intelligence Layer (InsightLayer)
A premium overlay panel providing three AI-driven cognitive analysis tools:
  • Velocity — Tracks how quickly your thinking is evolving, measuring the rate of new thought creation and connection formation
  • Contradictions — Detects conflicting thoughts within your graph, surfacing inconsistencies in your thinking
  • Gravity Wells — Identifies the "unasked questions" — important inquiries that your current thinking implies but that you haven't yet explored
Thought Maturity System
Each thought node has a maturity level that evolves as the thought is reinforced through multiple conversations:
SeedMentioned once, not yet developed
SproutingBeginning to grow connections
GrowingActively being developed and refined
MatureWell-established with multiple connections
EvolvedHas undergone significant revision or deepening
Stats Bar
A persistent status bar showing real-time metrics: total thought count, maturity breakdown distribution, and processing status during ingestion.
Section 18

Vault — Export & Import System

The Vault system ensures your knowledge is never locked in. Export your entire brain as structured markdown files, or import existing markdown/Obsidian vaults into Feynman.

Vault Export
Exports all knowledge nodes as a ZIP archive of structured markdown files. Each node becomes an individual .md file with frontmatter metadata (decay rate, lobe assignment, strength, connections). The system tracks the last export timestamp and node count, alerting users via the Command Menu when the vault is out of sync.
Vault Import
Import markdown files or Obsidian vaults into Feynman via the VaultImportModal. The import process parses markdown content, extracts frontmatter metadata, and creates knowledge nodes in the 3D brain with appropriate lobe mapping and connection detection.

Panel Management

The application enforces mutual exclusivity between major panels. Opening any feature (Chat, Study, Notes, Mindscape, Delphi, Alexander, Engram) automatically closes all other open panels. This prevents visual conflicts and ensures the user's full attention is devoted to one feature at a time.

The Command Menu itself closes on outside click or Escape key press, and includes an active panel indicator showing which feature is currently open.

Responsive Design

The entire platform uses the useResponsive() hook to provide adaptive layouts across devices:

  • Mobile — Full-screen panels, bottom sheet transitions, touch-optimized controls
  • Tablet — Reduced panel widths (320px vs 380px), adjusted padding
  • Desktop — Full-width panels, blur-heavy glassmorphism, hover effects
  • Touch Detection — Reduced backdrop-filter intensity on touch devices for performance
